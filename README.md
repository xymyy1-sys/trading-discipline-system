# 交易纪律系统

A 股短线/超短线个人交易纪律工具。第一版不接券商交易 API，不做真实交易下单，核心目标是识别当前最强题材、资金情绪、板块共振、板块龙头、连板标的、容量中军和题材炒作阶段，同时保留持仓/交易记录、买入前检查、集中进攻退出卡、卖出执行卡和复盘纪律。

## 技术栈

- 前端：React + TypeScript + Vite + ECharts
- 后端：FastAPI + SQLAlchemy + SQLite
- 行情数据：东方财富优先，新浪财经资金流备用，AkShare 作为辅助适配
- 部署：Docker Compose

## 当前已实现

- 题材雷达首页：先把行业/概念/风格板块归因到交易主线，再按主线聚合资金净流入、主力流入、板块涨幅、涨停扩散、核心股表现和共振证据，输出最强题材、阶段判断和操作提示。当前主线词典覆盖机器人/物理 AI、PCB/玻璃基板、CPO/光模块、创新药、电网设备/电力、MLCC/被动元件、存储芯片、商业航天、能源金属/小金属、贵金属/黄金等。
- 题材资金曲线：优先读取新浪财经板块/概念分时资金流，直接展示强势题材盘中净流入曲线；东方财富板块接口不可用时不再退回个股拼凑。
- 板块内部核心识别：把强势板块内个股分为情绪龙头、连板标的、容量中军、趋势核心和前排强势。
- 资金流证据页面：行业/概念/地域资金流，今日/5 日/10 日切换；今日资金流支持新浪板块排行与分时曲线兜底，资金流作为题材判断证据，不单独作为买点。
- 5 分钟缓存：题材雷达和资金流证据接口前后端均按 5 分钟 TTL 缓存；切换菜单返回不会重复抓取外部行情源，手动刷新会强制重新同步。
- 信息差页面：聚合东方财富快讯和新闻联播，映射 A 股相关板块，并结合资金流状态输出“资金已验证 / 等资金确认 / 资金流出”。
- 后端题材雷达 API：`/api/market/theme-radar`
- 后端资金流 API：`/api/market/sector-flow`
- 后端信息差 API：`/api/intel/daily`
- 市场分档 API：`/api/market/grade`
- 买入前检查 API：`/api/checks/pre-trade`
- 持仓、交易日志、退出卡、卖出计划基础 API。
- 标准短线和集中进攻核心仓位规则。

# 安全部署必读

系统包含真实持仓和交易记录，生产环境必须启用登录保护。部署前复制 `.env.example` 为 `.env`，并至少设置：

```env
AUTH_USERNAME=admin
AUTH_PASSWORD=使用独立且足够长的密码
AUTH_SECRET=使用密码生成器生成至少32位随机字符串
AUTH_COOKIE_SECURE=false
```

首次启用认证后，未登录请求只能访问 `/api/health` 和 `/api/auth/login`。不要把 `.env`、数据库文件或数据库备份提交到 GitHub。

推荐发布流程：

1. 备份服务器上的 SQLite 数据库和 `.env`。
2. `git pull --ff-only` 拉取已验收版本。
3. 执行 `alembic upgrade head`，禁止删除旧数据库后重建。
4. `docker compose up -d --build` 重建服务。
5. 验证登录、持仓读取、写操作、SSE 和数据库迁移。

`docker-compose.yml` 只公开前端端口 `5173`，后端 `8000` 仅供 Docker 内部网络访问。服务器防火墙也应关闭公网 `8000`。正式启用 HTTPS 后，将 `AUTH_COOKIE_SECURE` 改为 `true`。
