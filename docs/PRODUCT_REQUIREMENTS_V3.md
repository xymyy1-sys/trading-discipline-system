# 知行交易驾驶舱 V3.0 需求规格说明书

## 1. 产品定位

知行交易驾驶舱是一套面向 A 股短线交易的私人决策系统。核心不是预测涨跌，而是把“收盘基准预期—竞价验证—开盘确认—盘中证据—预期差—执行反馈—盘后校准”串成可审计闭环。

## 2. 目标能力与当前边界

本节描述产品目标，不等同于生产验收。实现状态统一拆分为：代码完成、生产验证、样本验证、外部条件阻塞；任何一层未完成时，不得把“有代码”表述成“策略有效”。

1. 全市场决策主屏：市场温度、题材强弱、资金迁移、持仓风险和待确认建议。
2. 固定观察池：昨日涨停、近期涨停及炸板承接两类候选；每日首次生成，剔除不递补，支持手动增删。
3. 持仓驾驶舱：持仓事实、预期管理、分钟量价、事件轨迹、利润保护、时间止损与做 T 约束。
4. 打板预案：由真实涨停天梯生成，并在次日经竞价、开盘和承接验证。
5. 新闻信息差：综合资讯、持仓相关资讯、原文链接、影响方向与关联标的。
6. AI 证据研判：仓库保留受纪律约束的持仓问答代码路径；外部模型生产配置、通用自动研判和评估集本阶段暂缓，不能作为已验收能力。
7. 钉钉风险通知：仓库保留连接器代码；第 4 章接入本阶段暂缓，未启用且未完成可用性验收。

## 3. AI 研判功能要求

### 3.1 入口

- 今日决策：AI 全局复核。
- 自动观察池：每只标的 AI 深度研判。
- 个股研判及持仓预期驾驶舱：当前股票 AI 深度研判。

### 3.2 缓存与重生成

- 首次点击调用模型并保存分析正文、模型、输入摘要、生成时间。
- 再次打开优先展示缓存，不重复计费。
- “重新生成”明确强制调用模型并覆盖最新缓存。
- 标的切换后读取对应缓存，不串用其他股票结果。

### 3.3 输出纪律

- 必须区分事实、推断、反向证据与数据缺口。
- 必须给出失效条件、关键价位和执行约束。
- 禁止承诺收益、虚构行情、替用户自动下单。
- API Key 只通过服务器环境变量保存，不入库、不进入 Git。

## 4. 钉钉机器人接入要求

### 4.1 用户申请步骤

1. 在钉钉建立一个仅供本人或交易团队使用的群。
2. 打开群设置—机器人—添加机器人—自定义机器人。
3. 安全设置选择“加签”，复制 Webhook 和加签密钥。
4. 在服务器 `.env` 配置：
   - `DINGTALK_ENABLED=true`
   - `DINGTALK_WEBHOOK=完整Webhook`
   - `DINGTALK_SECRET=SEC开头的加签密钥`
5. 重建后端容器，通过系统测试接口发送验证消息。

### 4.2 通知策略

- 推送：严重低于预期、减仓/退出要求、止损触发、实时证据链重大变化。
- 不推送：普通采样、重复状态、已发送过的同一建议。
- 消息包含股票、级别、状态、建议和登录系统查看完整证据的提示。

## 5. 后续优化需求

### P0：数据与可靠性

1. 建立数据源健康矩阵：每个接口展示更新时间、延迟、缺失率、降级源和交易日一致性。
2. 给行情、资金、新闻建立原始数据留痕，支持从原始响应重算指标。
3. 采集器增加任务队列、指数退避、熔断和告警，避免 SSE 中断被误解为业务故障。
4. 增加数据库自动备份、恢复演练和持仓/交易记录导出。

### P1：预期闭环

1. 将收盘基准预期拆成情景树：强修复、弱修复、延续、分歧、退潮，每种情景配置概率、区间和失效条件。
2. 保存每次竞价、开盘 5 分钟、午前、午后和收盘的预期修正，形成不可覆盖的版本链。
3. 建立“建议—用户确认—实际成交—结果”闭环，区分没看到、不同意、未成交和纪律违背。
4. 对建议执行覆盖更新，历史变化进入版本表，避免数据库建议数量膨胀。

#### 预期拥挤与反身性博弈层

该层用于识别“多数参与者对同一预期采取相似动作后，价格如何反过来改变其行为”的可观察路径。它不推断庄家、主力或散户的真实身份和主观意图，也不把单条评论、传闻或盘口现象直接写成确定性结论。

1. 市场级与个股级都必须同时评估以下四种可证伪路径；四种路径是竞争假设，不是固定剧本：
   - **反弹承接增强**：抛压被持续承接，价格收复并维持关键均价，首次回踩不破前低或支撑，回踩缩量，市场或板块广度同步改善。
   - **反弹失败、抛压延续**：反弹无法收复或站稳关键均价，量能跟不上，板块资金仍流出，反弹后再次走弱。
   - **无反弹、恐慌延续**：价格持续创新低，下跌时放量，跌停或大跌家数继续扩张，市场和板块没有修复证据。
   - **超预期修复、弱转强**：集合竞价或开盘显著好于基准预期，价格站稳关键均价，市场、板块与个股形成同向共振。
2. 市场级客观代理指标至少包括：涨跌家数与中位数涨幅、涨跌停比、大跌家数、全市场成交额及相对前一交易日/近五日比例、主力净流向、板块资金广度与集中度、主要指数相对均价的位置、前后快照的修复或恶化速度。
3. 个股级客观代理指标至少包括：集合竞价/开盘相对预期区间、实际与预期差、价格相对可靠 VWAP、日内低点反弹与高点回撤、量比与量能加速度、上攻效率、关键支撑是否有效、所属板块与全市场是否共振。
4. 每次评估必须输出：当前领先路径及其状态（假设/待确认/已确认/已失效）、支持证据、反向证据、数据缺口、下一验证点、失效条件、允许动作与禁止动作；不得只给“吸筹”“洗盘”“派发”等不可验证标签。
5. 若引用新闻、评论热度或社交情绪，只能作为“预期拥挤假设”的线索；在没有合规且具有代表性的样本、采集时间和覆盖范围时，必须标为未验证，不得声称市场共识已经形成。
6. 市场高风险只冻结扩仓、抄底和做 T 回补，不得仅因指数或情绪恶化就在日内极端低位机械卖出。执行顺序为：当日计划中的明确硬止损优先；未触发硬止损时，再依据反弹、承接、回踩和板块共振的后续证据决定减仓或等待确认。
7. 数据缺失、过期、时点不一致或 VWAP 不可靠时，结论必须降级为“证据不足/等待验证”，降低置信度并收紧允许动作；不得用默认值、模拟数据或模型猜测补齐关键证据。
8. 同一证据不得在多个相关指标中重复加权；负向证据必须允许被后续正向证据抵消，路径状态随新快照覆盖更新，完整变化写入不可覆盖的版本链。

#### 盘中量价、资金动能与增量方向

1. 板块与持仓资金必须保存带真实时点的连续快照，并计算净流入/净流出方向、每交易分钟流速、加速度和流入流出拐点；只有一个快照、跨交易日、午休穿越、未来时点或陈旧缓存时不得生成动能结论。
2. 量价语义必须区分：缩量上涨背离、缩量反弹未确认、缩量下跌抛压衰减、缩量下跌但资金仍弱、缩量回踩承接、放量下跌且资金加速流出、放量上涨/反弹确认。任何单一“缩量”或“放量”标签都不得直接生成买卖动作。
3. 下跌中资金拐入只进入反抽观察；须继续验证低点抬高、可靠分时均价修复和板块共振。上涨中资金拐出进入利润保护观察；须结合冲高回撤、均价线和既定计划确认，避免机械卖在最低点或盲目追高。
4. 盘中增量方向必须由最近交易分钟内新增多只真实涨停、板块实时涨幅、资金动能和可靠分时均价共同确认；输出核心/最早封板标的、风险和失效条件，只加入观察，不直接构成追板或买入信号。
5. 新闻必须分为正式公告、媒体归因和传闻，并保留标题、原文链接、来源和发布时间。资金量价同向只能验证“市场正在交易该影响”，不能把传闻升级为事实或断言单一因果。
6. 昨日深水修复后次日板块普遍高开、随后回落的“一致性兑现”只能在昨日修复、真实竞价广度、盘中回撤、可靠均价线及资金转弱均可验证时触发；关键证据缺失时输出数据缺口，不用默认值补齐。
7. 今日决策在交易时段每分钟巡检机会雷达和板块资金，数据源内部缓存可按接口能力降频；菜单切换不得强制刷新。手动刷新可重新拉取，但仍须展示数据时点和质量。

### P1：选股与组合

1. 观察池评分加入题材生命周期、相对板块强弱、筹码承接、炸板回封概率和风险收益比。
2. 增加相似标的去重：同题材仅保留龙头、换手核心和低位补涨各一只。
3. 增加组合暴露：行业、题材、风格、相关性和单一风险因子集中度。
4. 观察池记录进入原因、退出原因、观察天数和最终转化率。

### P2：模型校准

1. 按市场环境分别统计预期命中率、建议收益分布、最大不利波动和执行偏差。
2. 自动提出参数调整，但必须经人工确认后生效，并保留前后参数及回滚入口。
3. 建立 AI 分析评估集，检测幻觉、遗漏反向证据、结论漂移和过度交易倾向。
4. AI 输出采用结构化 JSON 后再渲染，支持字段级比较和回测。

### P2：安全与运维

1. 启用 HTTPS、域名、Secure Cookie、登录限速和可选双因素认证。
2. root 登录改为 SSH Key，关闭密码登录并限制管理端口来源。
3. API Key、钉钉密钥迁移到专用 Secret 管理，不在普通 `.env` 长期明文保存。
4. 增加发布回滚、容器日志轮转、磁盘监控和服务可用性通知。

## 6. 模拟盘与策略验证

### 6.1 产品边界

- 模拟盘是与真实持仓、真实交易日志完全隔离的“前向影子执行实验室”，不连接券商、不产生真实委托。
- 第一阶段只使用系统启用之后实际到达的数据做前向验证；在分钟行情、新闻和竞价历史数据未形成规范化时序库以前，不用收盘后完整数据冒充早盘可见数据做历史回测。
- 模拟结果用于比较规则和发现执行偏差，不自动修改实盘参数；参数晋级必须人工确认并保留回滚版本。

### 6.2 模拟账户与撮合

1. 支持打板、预期量价、持仓执行三类策略来源，记录模拟账户、委托、成交、持仓、现金、冻结资金和每日净值。
2. 信号在某分钟收盘后形成时，最早使用下一分钟实际可成交行情撮合；所有证据必须满足 `available_at <= decision_at`，禁止未来函数。
3. 执行 A 股 T+1、证券品种差异、涨跌停不可成交、停牌、最小委托单位、现金和可卖数量校验；无卖盘的一字涨停不得假设买入成功，无买盘的跌停不得假设卖出成功。
4. 收益必须扣除按证券类型配置的佣金、最低佣金、卖出印花税、过户费和滑点；缺失行情、陈旧行情或无法判断成交时标为无效样本，不用默认价格填充。
5. 模拟表与真实 `holdings`、`trade_logs` 完全隔离，任何模拟操作不得改变真实资产、持仓、当日盈亏和风险通知。

### 6.3 决策证据冻结

- 每个模拟决策冻结当时的市场环境、外围市场、板块资金、预期版本、分钟量价、事件时间线、策略版本、允许/禁止动作、失效/恢复条件、数据质量和原始证据引用。
- 保存决策时间、行情事件时间和系统收到时间；旧证据不可被后来的规则或数据覆盖。
- 同一证据哈希、同一策略版本和同一费用配置重复运行必须得到一致结果。

### 6.4 绩效与校准

- 基础指标：完整开平仓样本数、净胜率、平均盈利、平均亏损、盈亏比、期望值、利润因子、最大不利/有利波动、资金利用率、净值曲线和最大回撤。
- 分层指标：按策略、市场环境、题材/板块、预期差、交易时段和数据质量统计，并显示样本数；样本不足时不得标记“规则有效”。
- 打板专项：真实可成交率、封板/炸板率、次日开盘与收盘溢价、低开率、次日最大回撤、题材梯队完整度与身份角色。
- 预期量价专项：预期命中/证伪、错误卖出后反弹、未卖后继续回撤、冲高减仓捕获率、恐慌保护有效率、逆势加仓评估后的后续表现。
- 后续采用基准/挑战者并行和样本外验证；AI只解释结果，不自动调参或下单。

## 7. 验收原则

- 不以“页面存在”为完成，而以真实数据、可追溯来源、明确时点和可重复验证为准。
- 所有自动建议必须显示证据、反证、数据质量和失效条件。
- 预期拥挤与反身性评估必须能说明“为什么是当前路径、什么证据会改变判断、下一次何时验证”；不得把行为路径表述为已知参与者意图。
- 在同一市场风险快照下，系统必须区分“方向风险高”与“当前价位是否适合执行卖出”；未触发明确硬止损时，高风险不得直接等价为低位清仓。
- 所有外部写操作必须有权限控制和审计记录。
- 任何模型不可绕过交易纪律状态机直接生成下单动作。
