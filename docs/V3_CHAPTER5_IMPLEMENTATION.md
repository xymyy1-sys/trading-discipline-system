# 知行交易驾驶舱 V3.0 第 5 章实施记录

## 本批范围

- 暂缓第 3 章 AI 研判和第 4 章钉钉机器人。
- 无域名阶段暂不启用 HTTPS、Secure Cookie；保留配置入口。
- 量价与预期作为统一主链：收盘基准预期 → 情景树 → 集合竞价/开盘/盘中验证 → 量价证据 → 预期差 → 预期拥挤与反身性路径 → 执行建议 → 用户反馈 → 成交结果 → 复盘校准。

## 已实施

### P0 数据与可靠性

- 数据源健康矩阵：更新时间、延迟、缺失率、降级/陈旧、交易日一致性。
- 原始响应留痕、哈希校验，并为分钟行情提供从原始响应重算量价快照的接口。
- 采集器有限重试、指数退避、连续失败熔断、队列深度和熔断状态。
- SQLite 在线备份、完整性校验、保留策略、恢复脚本与恢复前备份。
- 持仓、交易记录 CSV 导出。

### P1 预期闭环

- 强修复、弱修复、延续、分歧、退潮五类情景树，包含概率、区间、验证条件、失效条件和纪律。
- 预期修正采用不可覆盖版本链；当前快照只作为读取加速层。
- 每个版本同时保存量价状态、VWAP、偏离、数据质量、正反证据和失效条件。
- 操作建议继续按标的覆盖更新，关键变化写入独立历史版本表。
- 执行反馈区分已执行、部分执行、不同意、未成交、没看到、纪律违背，并尝试关联真实成交。

#### 预期拥挤与反身性博弈层

- 市场与个股分别运行四条竞争路径：反弹承接增强、反弹失败抛压延续、无反弹恐慌延续、超预期修复弱转强；路径均配置验证条件和失效条件，不能以固定盈亏阈值替代证据判断。
- 市场路径使用涨跌广度、涨跌停比、大跌家数、成交额相对前日/近五日、主力净流向、板块资金广度与集中度、指数相对均价及前后快照变化。
- 个股路径使用竞价/开盘预期差、价格与可靠 VWAP、低点反弹、高点回撤、量能与上攻效率、关键支撑、板块及市场共振。
- 输出统一包含：领先路径与确认状态、支持证据、反向证据、数据缺口、下一验证点、失效条件、允许动作、禁止动作和置信度。
- “吸筹、洗盘、派发、主力意图”等只允许作为待验证行为假设；系统不得根据单条评论、传闻或一次盘口变化推断参与者真实意图。
- 市场高风险先冻结扩仓、抄底和做 T 回补，不直接生成日内低位清仓结论；明确的当日硬止损优先执行，未触发硬止损时等待反弹承接、回踩或板块共振的后续验证。
- 数据缺失、陈旧、时点不一致或 VWAP 不可靠时自动降级为“证据不足”，降低置信度并收紧允许动作，不使用模拟值补齐。
- 相关证据按证据家族去重，正反证据可动态抵消；当前路径随最新快照覆盖更新，状态变化进入版本链用于复盘校准。

#### 本层验收要点

- 任一路径都能展示其客观依据、反证、下一验证时点和失效条件。
- 相同证据输入必须产生可重复结果；缺少关键数据时不得输出确定性参与者意图。
- 同为高市场风险时，系统能区分“禁止新增风险”与“当前是否应卖出”，未触发硬止损不得因市场标签机械追卖。
- 后续快照出现反向证据时，领先路径和允许动作能够更新，并保留变更记录。

### P1 选股与组合

- 同题材按龙头、换手核心、低位补涨去重。
- 组合暴露按行业、题材、仓位风格、风险因子统计集中度并预警。
- 观察池记录进入/退出原因、观察天数、转持仓状态；用户剔除后当日不递补。

### P2 校准、安全与运维

- 按真实市场环境快照分层统计预期命中率、建议采纳率和平均不利波动。
- 参数调整仍须人工确认，保存前后快照并支持回滚。
- 密码、会话密钥及第三方密钥支持从 Secret 文件读取。
- 可选 TOTP 双因素认证。
- 容器日志轮转、磁盘/容器/接口巡检、发布回滚与 SSH 密钥化脚本。

## 本次业务修复

- “今日盈亏”改为“当前持仓今日浮动盈亏 + 今日卖出/清仓已实现盈亏”，并显示两部分构成。

## 暂缓或需要外部条件

- AI 评估集、结构化 AI 输出：随第 3 章暂缓。
- 钉钉可用性通知：随第 4 章暂缓；当前先由巡检脚本提供退出码。
- HTTPS、Secure Cookie：等待域名。
- 服务器关闭 SSH 密码登录：代码已提供安全脚本，但必须先取得并验证用户公钥，不能在没有可用密钥时直接执行，以免锁死服务器。
- 相关性矩阵需要积累足够的同频历史收益序列；当前先展示可证实的行业、题材、风格和风险因子暴露，不伪造相关系数。
